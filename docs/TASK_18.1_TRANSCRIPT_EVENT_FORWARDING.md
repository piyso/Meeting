# Task 18.1: Send Transcripts to Renderer via IPC

## Status: ✅ COMPLETE

## Overview

Implemented real-time transcript event forwarding from the main process (TranscriptService) to the renderer process via IPC. This enables the UI to display transcripts as they are generated by the ASR worker.

## Implementation Details

### 1. Main Window Export

**File:** `electron/main.ts`

Added `getMainWindow()` function to expose the main window instance to other modules:

```typescript
export function getMainWindow(): BrowserWindow | null {
  return mainWindow
}
```

This allows the IPC handlers to access the window's webContents for sending events.

### 2. Event Forwarding Setup

**File:** `src/main/ipc/handlers/transcript.handlers.ts`

Added `setupTranscriptEventForwarding()` function that:

1. Listens to 'transcript' events from TranscriptService
2. Transforms event data to TranscriptChunk format
3. Sends events to renderer via `webContents.send()`
4. Handles edge cases (window destroyed, window null)

**Key Features:**

- **Type-safe event handling**: Uses TranscriptChunk interface from IPC types
- **Graceful degradation**: Checks if window exists and is not destroyed before sending
- **Automatic transformation**: Converts TranscriptService event format to renderer-expected format
- **isFinal flag**: All saved transcripts are marked as final (isFinal: true)
- **Null handling**: Missing speakerId is converted to null

### 3. Event Flow

```
TranscriptService.saveTranscript()
  ↓
  emit('transcript', data)
  ↓
setupTranscriptEventForwarding() listener
  ↓
  Transform to TranscriptChunk
  ↓
  mainWindow.webContents.send('event:transcriptChunk', chunk)
  ↓
Renderer receives via window.electronAPI.on.transcriptChunk()
```

### 4. TranscriptChunk Interface

```typescript
interface TranscriptChunk {
  meetingId: string
  transcriptId: string
  text: string
  startTime: number
  endTime: number
  confidence: number
  speakerId: string | null
  isFinal: boolean
}
```

## Usage in Renderer

### Subscribe to Transcript Events

```typescript
// In React component
useEffect(() => {
  const unsubscribe = window.electronAPI.on.transcriptChunk(chunk => {
    console.log('New transcript:', chunk.text)
    console.log('Time:', chunk.startTime, '-', chunk.endTime)
    console.log('Confidence:', chunk.confidence)
    console.log('Speaker:', chunk.speakerId)
    console.log('Is final:', chunk.isFinal)

    // Update UI with new transcript
    setTranscripts(prev => [...prev, chunk])
  })

  return () => unsubscribe()
}, [])
```

### Example: Auto-scrolling Transcript Display

```typescript
function TranscriptDisplay({ meetingId }: { meetingId: string }) {
  const [transcripts, setTranscripts] = useState<TranscriptChunk[]>([])
  const scrollRef = useRef<HTMLDivElement>(null)

  useEffect(() => {
    const unsubscribe = window.electronAPI.on.transcriptChunk(chunk => {
      if (chunk.meetingId === meetingId) {
        setTranscripts(prev => [...prev, chunk])

        // Auto-scroll to bottom
        setTimeout(() => {
          scrollRef.current?.scrollTo({
            top: scrollRef.current.scrollHeight,
            behavior: 'smooth',
          })
        }, 100)
      }
    })

    return () => unsubscribe()
  }, [meetingId])

  return (
    <div ref={scrollRef} className="transcript-container">
      {transcripts.map(chunk => (
        <div key={chunk.transcriptId} className="transcript-chunk">
          <span className="timestamp">
            {formatTime(chunk.startTime)} - {formatTime(chunk.endTime)}
          </span>
          <span className="speaker">{chunk.speakerId || 'Unknown'}</span>
          <span className="text">{chunk.text}</span>
          <span className="confidence">{(chunk.confidence * 100).toFixed(0)}%</span>
        </div>
      ))}
    </div>
  )
}
```

## Error Handling

### Window Not Available

If the main window is null or destroyed, the event is not sent and a warning is logged:

```
[IPC] Cannot send transcript event: main window not available
```

This prevents crashes when:

- App is shutting down
- Window is being recreated
- Window hasn't been created yet

### Type Safety

All error handling uses proper TypeScript types:

```typescript
catch (error: unknown) {
  const errorMessage = error instanceof Error ? error.message : 'Unknown error'
  return {
    success: false,
    error: {
      code: 'TRANSCRIPT_GET_ERROR',
      message: errorMessage,
      timestamp: Date.now(),
    },
  }
}
```

## Performance Considerations

### Event Frequency

- Transcripts are saved in chunks (10-30 seconds of audio)
- Events are sent immediately after database save
- No buffering or throttling needed at this layer

### Memory Management

- Events are fire-and-forget (no response expected)
- No event queue or backlog
- Renderer is responsible for managing transcript state

### Logging

Each forwarded event is logged:

```
[IPC] Forwarded transcript event to renderer: Hello, this is a test trans...
```

This helps with debugging but can be disabled in production.

## Integration with ASR Worker

When the ASR worker completes transcription:

1. ASR Worker → Main Process: Sends transcript via IPC
2. Main Process → TranscriptService: Calls `saveTranscript()`
3. TranscriptService → Database: Saves to SQLite
4. TranscriptService → EventEmitter: Emits 'transcript' event
5. Event Handler → Renderer: Forwards via `webContents.send()`
6. Renderer → UI: Updates transcript display
